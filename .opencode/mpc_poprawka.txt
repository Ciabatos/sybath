Rozumiem — chcesz narzędzie get_function_definition, które zwróci pełną treść funkcji (jej ciało, nie tylko sygnaturę). Oto implementacja:
typescript// ── get_function_definition ───────────────────────────────────────────────────

const GetFunctionDefinitionInputSchema = z
  .object({
    function_name: z
      .string()
      .describe(
        "Name of the function to inspect, e.g. 'do_player_movement'. " +
          "Partial names are supported — if multiple functions match, all are returned."
      ),
    schema: z
      .string()
      .optional()
      .describe(
        "Filter by PostgreSQL schema, e.g. 'world', 'players'. " +
          "Omit to search across all schemas."
      ),
    response_format: z
      .enum(["json", "markdown"])
      .default("markdown")
      .describe("Output format. 'markdown' (default) or 'json'."),
  })
  .strict();

function registerGetFunctionDefinition(server: McpServer): void {
  server.registerTool(
    "get_function_definition",
    {
      title: "Get Function Definition (Full Source)",
      description: `Returns the full source code and metadata of one or more PostgreSQL functions/procedures.

Unlike get_functions or get_all_functions (which only return signatures), this tool
returns the complete function body — useful for understanding logic, argument handling,
SQL queries inside the function, error handling, etc.

Each result includes:
- schema            PostgreSQL schema
- function_name     Function name
- kind              'function' or 'procedure'
- language          Implementation language (plpgsql, sql, c, etc.)
- arguments         Full argument list string
- return_type       Return type string
- security          'definer' or 'invoker'
- volatility        'volatile', 'stable', or 'immutable'
- comment           SQL comment if set (api_type annotation), null if none
- source            Full function body (prosrc from pg_proc)
- definition        Complete CREATE OR REPLACE FUNCTION DDL statement

Args:
  - function_name (string): Name or partial name to search for (case-insensitive substring match).
  - schema (string, optional): Limit search to a specific schema.
  - response_format ('markdown' | 'json'): Output format (default: 'markdown')

Examples:
  - "Show me the source of do_player_movement"
    → call with function_name: 'do_player_movement'
  - "How does the inventory transfer function work internally?"
    → call with function_name: 'transfer', schema: 'inventory'
  - "What does get_known_map_tiles actually query?"
    → call with function_name: 'get_known_map_tiles'`,
      inputSchema: GetFunctionDefinitionInputSchema,
      annotations: {
        readOnlyHint: true,
        destructiveHint: false,
        idempotentHint: true,
        openWorldHint: false,
      },
    },
    async ({ function_name, schema, response_format }) => {
      try {
        const functions = await fetchFunctionDefinitions(function_name, schema ?? null);

        if (functions.length === 0) {
          return {
            content: [
              {
                type: "text",
                text:
                  `No function found matching name '${function_name}'` +
                  (schema ? ` in schema '${schema}'` : "") +
                  ".",
              },
            ],
          };
        }

        if (response_format === "json") {
          const json = JSON.stringify(functions, null, 2);
          return {
            content: [{ type: "text", text: truncateIfNeeded(json) }],
            structuredContent: { functions },
          };
        }

        const lines: string[] = [
          `# Function Definition${functions.length > 1 ? "s" : ""} — \`${function_name}\` (${functions.length} match${functions.length > 1 ? "es" : ""})`,
          "",
        ];

        for (const f of functions) {
          lines.push(`## ${f.schema}.${f.function_name}`);
          lines.push("");
          lines.push(`- **Kind:** ${f.kind}`);
          lines.push(`- **Language:** ${f.language}`);
          lines.push(`- **Arguments:** ${f.arguments || "—"}`);
          lines.push(`- **Returns:** ${f.return_type}`);
          lines.push(`- **Security:** ${f.security}`);
          lines.push(`- **Volatility:** ${f.volatility}`);
          lines.push(`- **API Comment:** ${f.comment ?? "—"}`);
          lines.push("");
          lines.push("### Full DDL");
          lines.push("```sql");
          lines.push(f.definition);
          lines.push("```");
          lines.push("");
        }

        return { content: [{ type: "text", text: truncateIfNeeded(lines.join("\n")) }] };
      } catch (err) {
        return {
          content: [
            {
              type: "text",
              text: `Error fetching function definition: ${err instanceof Error ? err.message : String(err)}`,
            },
          ],
          isError: true,
        };
      }
    }
  );
}

1. Typ w types.ts:
typescriptexport interface FunctionDefinitionInfo {
  schema: string;
  function_name: string;
  kind: "function" | "procedure";
  language: string;
  arguments: string;
  return_type: string;
  security: "definer" | "invoker";
  volatility: "volatile" | "stable" | "immutable";
  comment: string | null;
  definition: string;
}
2. Zapytanie w db.ts:
typescriptexport async function fetchFunctionDefinitions(
  name: string,
  schema: string | null
): Promise<FunctionDefinitionInfo[]> {
  const rows = await sql`
    SELECT
      n.nspname                                        AS schema,
      p.proname                                        AS function_name,
      CASE p.prokind
        WHEN 'p' THEN 'procedure'
        ELSE 'function'
      END                                              AS kind,
      l.lanname                                        AS language,
      pg_get_function_arguments(p.oid)                 AS arguments,
      pg_get_function_result(p.oid)                    AS return_type,
      CASE p.prosecdef
        WHEN true THEN 'definer'
        ELSE 'invoker'
      END                                              AS security,
      CASE p.provolatile
        WHEN 'v' THEN 'volatile'
        WHEN 's' THEN 'stable'
        WHEN 'i' THEN 'immutable'
      END                                              AS volatility,
      obj_description(p.oid, 'pg_proc')               AS comment,
      pg_get_functiondef(p.oid)                        AS definition
    FROM pg_proc p
    JOIN pg_namespace n ON n.oid = p.pronamespace
    JOIN pg_language  l ON l.oid = p.prolang
    WHERE p.proname ILIKE ${'%' + name + '%'}
      AND n.nspname NOT IN ('pg_catalog', 'information_schema')
      ${schema ? sql`AND n.nspname = ${schema}` : sql``}
    ORDER BY n.nspname, p.proname
  `;
  return rows as FunctionDefinitionInfo[];
}

Kluczowe elementy tego rozwiązania:

pg_get_functiondef(p.oid) — zwraca pełne DDL z CREATE OR REPLACE FUNCTION włącznie z ciałem
p.prosrc byłby samym body, ale pełne DDL jest czytelniejsze dla LLM
Wyszukiwanie po podciągu (ILIKE '%name%') pozwala znaleźć funkcję nawet bez dokładnej nazwy
security i volatility to dodatkowy kontekst przydatny przy analizie zachowania funkcji
